Custom Validation.

Angular provides basic implementation for most common HTML5 input types: (text, number, url, email, date, radio, checkbox),as well as some directives for validation (required, pattern, minlength, maxlength, min, max).With a custom directive, you can add your own validation functions to the $validators object on the ngModelController.

To get a hold of the controller, you require it in the directive as shown inthe example below.Each function in the $validators object receives the modelValue and the viewValue as parameters. Angular will then call $setValidity internally with the function's return value (true: valid, false: invalid).

The validation functions are executed every time an input is changed ($setViewValue is called)or whenever the bound model changes. Validation happens after successfully running $parsers and $formatters, respectively.Failed validators are stored by key in ngModelController.$error.
